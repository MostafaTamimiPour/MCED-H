# Mixed-Criticality Scheduling for Energy Harvesting Real-Time Systems

A Python implementation of multiple scheduling algorithms for energy harvesting mixed-criticality real-time systems, including EDF, EDF-VD, EDH, MCQPA, and the proposed MCED-H algorithm.

---

# Project Overview

This repository presents a Python implementation of scheduling algorithms for energy harvesting mixed-criticality real-time systems. The project investigates how different scheduling strategies perform under limited and dynamically harvested energy while satisfying the timing requirements of tasks with different criticality levels.

The repository includes implementations of several well-known scheduling algorithms, including EDF, EDF-VD, EDH, and MCQPA, together with MCED-H, the scheduling algorithm proposed in my master's research.

To enable fair and reproducible evaluations, the project also provides a configurable task-set generator, common utility modules, experiment runners, and plotting scripts for reproducing the simulation results.

The implementation is designed to facilitate performance comparison among scheduling algorithms using metrics such as acceptance ratio under different system utilization levels.

# Implemented Algorithms

The repository includes implementations of the following scheduling algorithms for mixed-criticality energy harvesting real-time systems.

| Algorithm | Description |
|-----------|-------------|
| **EDF** | Earliest Deadline First scheduling. |
| **EDF-VD** | Earliest Deadline First with Virtual Deadlines for mixed-criticality systems. |
| **EDH** | Energy-Driven Heuristic scheduling algorithm. |
| **MCQPA** | Mixed-Criticality Queue-based Priority Assignment scheduling algorithm. |
| **MCED-H** | Proposed scheduling algorithm developed in this research for energy harvesting mixed-criticality real-time systems. |

Among these algorithms, **MCED-H** is the scheduling algorithm proposed as part of this research, while the remaining algorithms are implemented as baseline methods for performance comparison.

# Task Set Generation

Task sets are generated using a custom point-based task generator designed for mixed-criticality energy harvesting real-time systems.

The generator produces two files:

- **Tasks.txt** — contains the complete specification of every generated task set.
- **Utilizations.txt** — stores the utilization values corresponding to each generated task set.

The generated datasets are then used by all scheduling algorithms to ensure a fair and reproducible comparison.

# How to Run

## 1. Generate Task Sets

The task generation parameters (e.g., the number of task sets and other generation settings) can be configured in:

```text
Generators/PointGenerator/TaskGenerator.py
```

After configuring the desired parameters, generate the task sets by running:

```bash
python Generators/PointGenerator/TaskGenerate.py
```

This creates the following files:

- `Tasks.txt`
- `Utilizations.txt`

---

## 2. Run the Scheduling Algorithms

Execute any of the implemented scheduling algorithms:

```bash
python Runners/RunEDF.py
python Runners/RunEDFVD.py
python Runners/RunEDH.py
python Runners/RunMCEDH.py
python Runners/RunMCQPA.py
```

Each runner reads the generated task sets and stores the corresponding evaluation results in:

```text
Results/Raw/
```

---

## 3. Plot the Results

After running the algorithms, execute the plotting scripts to reproduce the evaluation figures:

```bash
python Results/Figures/PlotOverview.py
python Results/Figures/PlotLo.py
python Results/Figures/PlotHi.py
```
